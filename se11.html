User Interface Design

Creating Effective and Usable Software

The Importance of the User Interface

Why UI Design Matters

The User Interface (UI) is the bridge between the user and the computer system. Its design is critical for the success of any software project.

  • Users often judge a system by its interface rather than its underlying functionality.
  • A poorly designed interface can cause users to make catastrophic errors.
  • Poor UI design is a primary reason why many software systems are never used.
  • The mantra is: Keep it simple! Low usability leads to user errors, data corruption, and ultimately, system rejection.

Who Designs the UI?

While software engineers are involved, effective interface design is a multidisciplinary field requiring specialists like Human-Computer Interaction (HCI) experts, human factor specialists, and interface architects.

UI Design Principles

Foundations of Good Design

Good UI design takes into account the needs, experience, and capabilities of its users. It acknowledges human limitations and anticipates user mistakes.

Principle Description
User Familiarity Use terms and concepts from the user's domain, not computer-centric jargon (e.g., 'folders' and 'documents' instead of 'directories' and 'files').
Consistency Comparable operations should look and act the same way. Menus, commands, and layouts should be consistent throughout the application.
Minimal Surprise The behavior of the system should be predictable. Users should not be surprised by the outcome of an action.
Recoverability Provide mechanisms to recover from errors, such as 'undo', confirmation for destructive actions, and checkpointing.
User Guidance Provide meaningful feedback, help systems, and on-line manuals to guide the user.
User Diversity Support different types of users. For example, provide keyboard shortcuts for experts and accessible options (like larger text) for users with visual difficulties.

The UI Design Process

An Iterative Approach

UI design is not a linear process. It involves a continuous cycle of analysis, prototyping, and evaluation with close collaboration between designers and end-users.

The 3 Core Activities:

1. User Analysis

Understand who the users are and what they need to do with the system. Without this, designing an effective interface is impossible. Techniques include:

  • Task Analysis: Modeling the steps involved in completing a task.
  • Interviews & Questionnaires: Asking users about their work.
  • Ethnography: Observing users in their natural work environment.

2. UI Prototyping

Develop a series of prototypes to allow users to gain direct experience with the interface. This is crucial for judging usability.

  • Paper Prototypes: Early-stage sketches and storyboards to get quick feedback on concepts.
  • Automated Prototypes: Increasingly sophisticated interactive mock-ups using tools like Figma, Visual Basic, or web technologies.

3. Interface Evaluation

Experiment with the prototypes with real users to assess the design's suitability and identify problems.

  • Usability Testing: Observing users as they attempt to complete tasks with the prototype.
  • Questionnaires: Gathering subjective feedback from users.
  • Online Feedback: Collecting data directly from within the software.

Information Presentation

Displaying Data Effectively

This area of UI design is concerned with presenting system information to users in a clear and understandable way. Information can be static (doesn't change during a session) or dynamic (changes frequently).

Use of Color

  • Use for highlighting and showing relationships (e.g., layers).
  • Guidelines: Limit the number of colors, use color consistently, and avoid poor color pairings.
  • Crucially: Do not rely on color alone to convey meaning, as 10% of men and 2% of women are color-blind.

Use of Fonts

  • Serif fonts (like Times New Roman) are generally better for reading large blocks of text.
  • Sans-serif fonts (like Arial) are generally better for headings and presenting information on screens.

Error Messages

Error message design is critically important. Poor messages frustrate users and can lead them to reject a system.

Good Error Messages Are:

  • User-Oriented: Use plain language, not system-specific error codes like "Error #27".
  • Polite & Constructive: Be polite, avoid beeps/noises, and suggest how to correct the error.
  • Concise: Give enough information without being overwhelming.
  • Positive & Active: Frame messages positively and use an active voice.

Bad: "Error #27: Invalid patient id entered."

Good: "Patient J. Bates is not registered. Click 'Patients' for a list of registered patients or click 'Retry' to re-enter the name."

Interface Evaluation

Assessing Usability

Some form of evaluation must be carried out to assess a UI's suitability. This involves measuring key usability attributes.

Attribute Description
Learnability How long does it take a new user to become productive with the system?
Speed of Operation How well does the system response time match the user's work practices?
Robustness How tolerant is the system of user error?
Recoverability How good is the system at recovering from user errors?
Adaptability How easily can the system be tailored to individual user workflows?
