Question 3(c)

Write C++ or Python code to model the segment of the sequence diagram given below.

: Student
: Subject
get_subject_info()
Subject_details

Theoretical Deep Dive: Sequence Diagrams to Code

Sequence diagrams visualize interactions between objects in a system over time. They depict objects and the messages exchanged between them. Translating this to code involves:

  • Lifelines to Objects: Each lifeline (the vertical dashed line with a box at the top) represents an instance of a class. We need to define the `Student` and `Subject` classes.
  • Messages to Method Calls: A message sent from one lifeline to another represents a method call. The `Student` object must have a method that calls a method on the `Subject` object.
  • Message Name to Method Name: The label on the message arrow (`get_subject_info()`) becomes the name of the method being called on the `Subject` class.
  • Return Message to Return Value: The dashed return arrow (`Subject_details`) represents the value returned by the method call. The `get_subject_info` method on the `Subject` class must return an object or data structure containing the subject's details.

Model Answer (Python)


# In a real system, SubjectDetails would likely be a data class or similar structure.
from typing import NamedTuple

class SubjectDetails(NamedTuple):
    subject_code: str
    subject_name: str
    credits: int

class Subject:
    def __init__(self, code: str, name: str, credits: int):
        self._details = SubjectDetails(subject_code=code, subject_name=name, credits=credits)

    # This method is called by the Student object. It prepares and returns the details.
    def get_subject_info(self) -> SubjectDetails:
        # This method corresponds to the activation on the Subject lifeline.
        # It processes the request and returns data.
        print("Subject: Preparing and returning subject details...")
        return self._details

class Student:
    def __init__(self, name: str):
        self._name = name
        self._retrieved_details = None

    # This method initiates the interaction shown in the sequence diagram.
    def request_info_from_subject(self, subject_object: Subject):
        print(f"Student '{self._name}': Sending message 'get_subject_info()' to Subject.")
        
        # The line below is the actual message send: Student calls a method on Subject.
        details = subject_object.get_subject_info()
        
        # The code after the call represents the Student processing the return message.
        self._retrieved_details = details
        print(f"Student '{self._name}': Received return message with details: {self._retrieved_details}")

# --- Example Usage ---
# Create instances of the objects (lifelines in the diagram).
student1 = Student("Alice")
subject1 = Subject("COM2308", "Software Engineering", 3)

# The Student object initiates the sequence of messages.
student1.request_info_from_subject(subject1)
