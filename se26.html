Section 1: UI Fundamentals

  1. 1. What are the three essential design components of any computer system?

    Hardware design, Software design, and Interface design.

  2. 2. Who is typically responsible for interface design in most organizations?

    While software engineers are heavily involved, it is a multidisciplinary field requiring specialists like HCI experts, human factor specialists, and interface architects.

  3. 3. Why is good user interface design critically important?

    Because users often judge a system by its interface, a poor UI can cause catastrophic errors, and it is a primary reason why many software systems are never used.

  4. 4. What is the most important mantra for user interface design?

    Keep it simple!

  5. 5. What is the consequence of low usability?

    Low usability leads to user errors and misunderstandings, which can cause data corruption and ultimately lead to users rejecting the system.

  6. 6. Name the two main types of user interfaces (UIs).

    Command line interfaces and Graphical User Interfaces (GUIs).

  7. 7. Which type of UI is more popular for interactive systems?

    Graphical User Interfaces (GUIs).

  8. 8. List the five main characteristics of a GUI.

    Windows, Icons, Menus, Pointing devices, and Graphics.

  9. 9. What is the purpose of 'Windows' in a GUI?

    To display different information simultaneously and allow users to switch between tasks.

  10. 10. What is the purpose of 'Icons' in a GUI?

    To represent different data types or actions visually.

  11. 11. What is the purpose of 'Menus' in a GUI?

    To provide a structured way for users to select commands.

  12. 12. What is the purpose of 'Pointing devices' in a GUI?

    To allow users to select items and access any point on the screen directly.

  13. 13. List three main advantages of GUIs.

    1. They are easy to learn and use. 2. Windows help users to switch between tasks. 3. Pointing devices give fast access to any point on the screen.

  14. 14. What is a 'user-centered design' approach?

    A design philosophy that emphasizes the heavy involvement of end-users throughout the design process, both as evaluators and as members of the design team.

Section 2: Human Factors & Design Principles

  1. 15. Why is 'limited short-term memory' a critical human factor in UI design?

    Because people can only remember about 7 items of information at once. Presenting more than this increases cognitive load and makes users more liable to make mistakes.

  2. 16. How should a UI be designed to account for the fact that 'people make mistakes'?

    The UI should be designed to be forgiving. It should provide mechanisms for recovery (like 'undo') and use clear, calm messages to reduce stress when errors occur.

  3. 17. How should designers account for the fact that 'people are different'?

    Designers should not just design for their own capabilities. They must account for a wide range of physical abilities (e.g., provide accessibility options) and different interaction preferences (e.g., offering both graphical and textual ways to perform tasks).

  4. 18. What is the core idea behind all UI design principles?

    That UI design must take into account the needs, experience, and capabilities of the system users, and be aware of human limitations.

  5. 19. Explain the principle of 'User Familiarity'.

    The interface should use terms and concepts drawn from the user's real-world experience, not computer-oriented concepts (e.g., an office system should use 'folders' and 'documents', not 'directories' and 'files').

  6. 20. Explain the principle of 'Consistency'.

    Wherever possible, comparable operations should be activated in the same way. Menus, commands, and visual elements should have the same format and behavior throughout the system.

  7. 21. Explain the principle of 'Minimal Surprise'.

    Users should never be surprised by the behavior of a system. If a command operates in a known way, the user should be able to predict the operation of comparable commands.

  8. 22. Explain the principle of 'Recoverability'.

    The interface should include mechanisms to allow users to recover from errors. This might include an 'undo' facility, confirmation of destructive actions, or checkpointing.

  9. 23. Explain the principle of 'User Guidance'.

    The interface should provide meaningful feedback when errors occur and provide context-sensitive user help facilities and on-line manuals.

  10. 24. Explain the principle of 'User Diversity'.

    The interface should provide appropriate interaction facilities for different types of users. For example, offering keyboard shortcuts for experts and larger text for users with seeing difficulties.

Section 3: Interaction Styles & Information Presentation

  1. 25. List the five main interaction styles.

    Direct manipulation, Menu selection, Form fill-in, Command language, and Natural language.

  2. 26. What is 'Direct Manipulation'? Give an example application.

    An interaction style where users interact with visual objects on the screen, providing fast and intuitive interaction. Examples: Video games, CAD systems, dragging a file into a folder.

  3. 27. What is a major disadvantage of 'Menu Selection'?

    It can be slow for experienced users who know the commands they want to use.

  4. 28. For what type of task is 'Form Fill-in' most suitable?

    For simple data entry tasks. Examples: Stock control, personal loan processing.

  5. 29. What is the main advantage of a 'Command Language' interface?

    It is powerful and flexible, allowing experienced users to perform complex tasks quickly and create scripts for automation.

  6. 30. What is a major disadvantage of 'Natural Language' interfaces?

    Natural language understanding systems are often unreliable and can misinterpret the user's intent.

  7. 31. What is 'information presentation' concerned with?

    Presenting system information to system users in a clear and effective way, either directly (text) or transformed (graphics).

  8. 32. What is the Model-View-Controller (MVC) approach?

    An architectural pattern that separates the data model, the user interface (view), and the control logic. It is a way of supporting multiple presentations (views) of the same data.

  9. 33. When presenting information graphically, list four factors to consider.

    1. The need for precise information vs. relationships. 2. The dynamicity of the values. 3. The required user response to information changes. 4. Whether the information is textual or numerical.

  10. 34. What are the disadvantages of presenting information textually?

    It cannot be read "at-a-glance" and is not efficient for showing trends or changes in dynamic data.

  11. 35. Differentiate between 'static' and 'dynamic' information.

    • Static: Initialized at the beginning of a session and does not change (e.g., user's name).
    • Dynamic: Changes during a session and these changes must be communicated to the user (e.g., a progress bar).

  12. 36. How is dynamic data best represented?

    Graphically, using representations like dials, pie charts, thermometers, or horizontal bars.

  13. 37. Why is an analogue representation often better than a digital one for showing relative values?

    An analogue representation (like a horizontal bar) makes it easy to see the value's position within its possible range at a glance, which is harder to do with just a digital number.

  14. 38. What is the most important rule when representing numerical values in a list?

    Good alignment. Numbers should be right-aligned (or aligned on the decimal point) to make them easy to compare.

  15. 39. List three guidelines for using color in interfaces.

    1. Limit the number of colors. 2. Use color consistently. 3. Avoid poor color pairings (e.g., red on blue).

  16. 40. What is the critical accessibility reason for not using color as the sole means of conveying information?

    Because a significant portion of the population (10% of men, 2% of women) is color-blind.

  17. 41. Differentiate between serif and sans-serif fonts and their best uses.

    • Serif (splayed ends, e.g., Times New Roman): Good for reading large blocks of printed text.
    • Sans-serif (squared ends, e.g., Arial): Good for presenting information on computer screens.

  18. 42. What are the two types of user help requests?

    1. "Help!" (I need help, I'm stuck). 2. "Help?" (I need more information).

  19. 43. List three forms of user help and guidance.

    1. System messages responding to user actions. 2. Online help systems. 3. System documentation.

  20. 44. List three characteristics of good error messages.

    They should be polite, concise, consistent, and constructive.

  21. 45. What is the difference between a system-oriented and a user-oriented error message?

    A system-oriented message uses technical jargon and error codes (e.g., "Error #27: Invalid patient id"). A user-oriented message uses plain language, explains the problem in the user's context, and suggests a solution (e.g., "Patient J. Bates is not registered. Click Retry to re-input a patient name.").

Section 4: The UI Design Process

  1. 46. Is the UI design process linear or iterative?

    It is an iterative process involving close liaisons between users and designers.

  2. 47. What are the three core activities in the UI design process?

    1. User analysis. 2. System prototyping. 3. Interface evaluation.

  3. 48. What is the goal of 'user analysis'?

    To understand what the users will do with the system.

  4. 49. What is the goal of 'system prototyping'?

    To develop a series of prototypes for users to experiment with.

  5. 50. What is the goal of 'interface evaluation'?

    To experiment with these prototypes with users to assess their suitability.

  6. 51. Describe the flow of the UI design process as shown in the diagram.

    It is a cycle: Analyse user activities -> Produce paper prototype -> Evaluate with users -> Design dynamic prototype -> Evaluate with users -> Produce executable prototype -> Implement final UI. Evaluation feedback loops back into the design stages.

Section 5: User Analysis & Prototyping

  1. 52. Why is user analysis fundamental to effective interface design?

    If you don't understand what the users want to do with a system, you have no realistic prospect of designing an effective interface.

  2. 53. What is a 'scenario' in user analysis?

    A narrative description of a typical episode of use, which helps describe the user's goals and context in an understandable way.

  3. 54. List three analysis techniques for understanding users.

    1. Task analysis. 2. Interviewing and questionnaires. 3. Ethnography.

  4. 55. What is 'task analysis'?

    A technique that models the steps involved in completing a task, often hierarchically.

  5. 56. What is 'ethnography'?

    A technique that involves an external observer watching users at work and questioning them in an unscripted way.

  6. 57. Why is ethnography particularly valuable?

    Because many user tasks are intuitive, and users find them very difficult to describe and explain. Observation reveals what they actually do, not just what they say they do.

  7. 58. What is the main aim of user interface prototyping?

    To allow users to gain direct experience with the interface, as it is impossible to judge usability without it.

  8. 59. What is 'paper prototyping'?

    Working through scenarios using sketches of the interface, often with a storyboard to present a series of interactions.

  9. 60. What is 'script-driven prototyping'?

    Developing a set of predefined scripts and screens. When the user interacts (e.g., clicks a button), the screen changes to the next pre-scripted display.

  10. 61. What is 'visual programming' for prototyping?

    Using a language and environment designed for rapid development of GUIs, such as Visual Basic.

Section 6: Interface Evaluation

  1. 62. Why is some form of evaluation necessary?

    To assess the suitability and usability of a user interface design.

  2. 63. Why is full-scale evaluation often impractical?

    It is very expensive and time-consuming for most systems.

  3. 64. Ideally, what should an interface be evaluated against?

    A usability specification, though it is rare for such formal specifications to be produced.

  4. 65. List the five main usability attributes.

    Learnability, Speed of operation, Robustness, Recoverability, and Adaptability.

  5. 66. Define the 'Learnability' attribute.

    How long does it take a new user to become productive with the system?

  6. 67. Define the 'Speed of operation' attribute.

    How well does the system response match the user's work practice?

  7. 68. Define the 'Robustness' attribute.

    How tolerant is the system of user error?

  8. 69. Define the 'Recoverability' attribute.

    How good is the system at helping a user recover from errors?

  9. 70. Define the 'Adaptability' attribute.

    How closely is the system tied to a single model of work? Can it be adapted to different workflows?

  10. 71. List four simple evaluation techniques.

    1. Questionnaires for user feedback. 2. Video recording of system use. 3. Direct observation of users. 4. Collecting on-line user feedback via code in the software.

  11. 72. What is the advantage of video recording system use?

    It allows for detailed, post-session analysis of user behavior, hesitations, and errors that might be missed during live observation.

  12. 73. What is the purpose of collecting on-line user feedback?

    To gather quantitative data on how the interface is actually used, such as which features are used most frequently or where users most often encounter errors.

  13. 74. Why are open-ended questions useful in interviews and questionnaires?

    They allow users to provide information that they think is essential, not just information that the designer has thought of asking about.

  14. 75. What is the overall message of the "User Interface Design" presentation?

    That user interface design is a critical, iterative, and user-centered process. A successful UI must be built on sound design principles, an understanding of human factors, and be continuously refined through prototyping and evaluation with real users.

  15. 76. What is the main drawback of 'form fill-in' interfaces?

    They take up a lot of screen space and can cause problems when user options do not neatly match the available form fields.

  16. 77. What is the main drawback of 'direct manipulation' interfaces?

    They can be hard to implement and are only suitable where there is a clear visual metaphor for the tasks and objects involved.

  17. 78. How can a GUI help with a user's limited short-term memory?

    By making operations and options visible (e.g., in menus and toolbars), the interface relies on recognition rather than recall, reducing the memory load on the user.

  18. 79. In the MVC pattern, what is the role of the 'Model'?

    The Model manages the application's data and business logic. It is independent of the user interface.

  19. 80. In the MVC pattern, what is the role of the 'View'?

    The View is the presentation of the data from the Model in a particular format (e.g., a chart, a table).

  20. 81. In the MVC pattern, what is the role of the 'Controller'?

    The Controller handles user input and translates it into commands for the Model or View.

  21. 82. Give an example of why it's important to have good alignment for numerical values.

    With poor alignment, it is very difficult to quickly compare numbers. For example, it's hard to tell at a glance if 120 is larger than 75 if they are not right-aligned.

  22. 83. What is a 'storyboard' in the context of paper prototyping?

    A series of sketches or panels that show the sequence of interactions a user has with the interface to complete a task.

  23. 84. Why would you use a focus group or group interview for user analysis?

    It allows users to discuss their work with each other, which can reveal shared tasks, workflows, and pain points that might not come up in a one-on-one interview.

  24. 85. What does "context-sensitive user help" mean?

    The help system provides information that is directly relevant to the specific task the user is currently performing or the part of the interface they are interacting with.

  25. 86. What is the main benefit of a command line interface?

    It is powerful, flexible, and fast for expert users, and it can be easily scripted to automate repetitive tasks.

  26. 87. In the user interaction scenario, what is the primary problem Jane faces?

    She doesn't know the appropriate search terms to find the materials she needs. This highlights a requirement for the system to assist users in choosing search terms.

  27. 88. From the scenario, list two specific requirements for the library search system.

    1. Users need to be able to select specific collections to search. 2. Users need to be able to request copies of relevant material.

  28. 89. What does 'hierarchical task analysis' break down?

    It breaks down a high-level user goal (e.g., "Retrieve pictures") into a hierarchy of smaller, more detailed sub-tasks and the plans for executing them.

  29. 90. What is one reason a bar chart is more effective than text for dynamic data?

    A bar chart allows for "at-a-glance" comparison and trend-spotting, which is very difficult to do with a table of changing numbers.

  30. 91. What does the "positive, not negative" guideline for error message style mean?

    It means phrasing messages constructively. Instead of "You did not enter a valid date," say "Please enter the date in DD/MM/YYYY format."

  31. 92. What does the "active, not passive" guideline for error messages mean?

    Using an active voice is more direct and clear. For example, "The system will save your file" is better than "Your file will be saved by the system."

  32. 93. Why should error messages not be insulting or funny?

    Because a user encountering an error is already likely frustrated. Insulting or humorous messages can increase this frustration and make the software seem unprofessional.

  33. 94. What is the difference between fixed-pitch and variable-pitch fonts?

    In a fixed-pitch font (like Courier), every character takes up the same amount of horizontal space. In a variable-pitch font (like Times New Roman), characters have different widths (e.g., 'i' is narrower than 'm').

  34. 95. When would you use a fixed-pitch font?

    It is primarily used for displaying code or tabular data where vertical alignment of characters is important.

  35. 96. What is the main benefit of an 'undo' facility?

    It directly supports the 'Recoverability' principle, giving users the confidence to experiment and perform actions without fear of making irreversible mistakes.

  36. 97. What is the danger of designers only designing for their own capabilities?

    They are likely to be expert users with full physical capabilities. This can lead to an interface that is too complex for novice users and inaccessible to users with disabilities.

  37. 98. How can providing "on-line user feedback" be implemented?

    By including code in the software to log user actions (with their permission), track common errors, or provide a simple "Send Feedback" button.

  38. 99. What is the most crucial takeaway about the UI design process?

    That it must be iterative and involve close collaboration with end-users at every stage.

  39. 100. If a system has low usability, what is the ultimate consequence?

    Users will reject the system.
