What is Software?

According to Sommerville, software is not just the code. It encompasses:

Computer Programs

The executable code that delivers functionality.

Associated Documentation

User manuals, technical guides, and API docs.

Configuration Data

Settings and files needed for the program to operate correctly.

Generic Software

Stand-alone software products sold on the open market.

Examples:

  • Spreadsheet applications (e.g., Excel)
  • Database systems (e.g., MySQL)
  • Text editors (e.g., VS Code)

Bespoke Software

Software developed for a specific customer or purpose.

Examples:

  • Business process automation tools
  • Embedded systems software (in cars, appliances)
  • Air traffic control systems

System Software

Low-level software required to manage computer resources and support application execution.

Examples:

  • Operating Systems (Windows, Linux)
  • Database Management Systems (DBMS)
  • Network Management Software
  • Development Tools (Compilers, Debuggers)

Application Software

Software that performs a specific function directly for the end user.

Examples:

  • Word Processors & Spreadsheets
  • Games & Media Players
  • CRM & HR Management Software
  • Anti-virus & Utility Software

Food for Thought

Consider the software development community in your region (e.g., Sri Lanka). How many companies exist? How many people are employed? Are they primarily developing generic or bespoke software? This helps connect these concepts to the real world.

The History: Software Crisis

The term 'software engineering' was proposed in 1968 as a response to the "software crisis". As computer hardware advanced rapidly, software became more complex, leading to systemic problems:

  • Projects were frequently over budget and behind schedule.
  • Software was often of low quality and did not meet user requirements.
  • Projects were unmanageable, and the code was difficult to maintain.
  • Some projects were never delivered at all.

Software Engineering was proposed as the solution—a disciplined, engineering-based approach to tackle this complexity.

Computer Science

Focuses on the theory and fundamentals of computation, algorithms, data structures, and the underlying principles of computing.

Software Engineering

Focuses on practically producing software by applying computer science principles, along with engineering discipline, project management, and other ad hoc approaches to build reliable and maintainable systems.

The Software Process & Models

Core Software Process Activities

The software process is a structured set of activities required to develop a software system. It is generally broken down into four key phases:

1. Specification

Defining what the system should do. Includes requirements analysis and specification.

2. Development

Designing and building the software. Includes design, implementation, and documentation.

3. Validation

Ensuring the software does what the customer wants. Includes testing and debugging.

4. Evolution

Changing the software in response to changing demands. Includes maintenance.

Software Development Life Cycle (SDLC)

The SDLC is a series of steps or phases in which software is produced. The goals of each phase are to:

  • Mark out a clear set of steps to perform.
  • Produce a tangible item (e.g., a document, a prototype).
  • Allow for review of work.
  • Specify actions for the next phase.

Software Process Models

A software process model is a simplified representation (or abstraction) of a software process. Popular paradigms include:

  • Waterfall Model: Represents process activities as separate, sequential phases.
  • Iterative Development: "Cycles" through development phases, delivering functionality incrementally.
  • Component-based: Software is developed by integrating existing components.

CASE Tools

Computer-Aided Software Engineering (CASE) tools are software systems that support software process activities.

Upper-CASE Tools

Support early phases of the SDLC, such as requirements analysis and system design.

Lower-CASE Tools

Support later phases, such as implementation, testing, and maintenance.

Software Attributes & Objectives

Good software engineering aims to produce software that is not just functional, but also possesses a range of desirable qualities. These are the key objectives and attributes of well-engineered software.

Maintainability

The ability for the software to evolve to meet changing needs. This includes fixing bugs and adding new features.

Dependability & Reliability

The software should be reliable, safe, and secure. It should not cause physical or economic damage in the event of system failure.

Efficiency

The software should not waste system resources such as memory (RAM) or processor cycles (CPU time).

Usability

The ability for users to use the software easily and without hassle. It should have an appropriate user interface and adequate documentation.

Correctness & Testability

The software should perform according to its specification. It must be testable to verify this correctness.

Reusability & Portability

Components should be reusable in other systems. The software should be able to run on different hardware and operating systems.

Key Challenges in Software Engineering

The Legacy Challenge

Many valuable software systems were developed long ago using outdated technologies. Maintaining and evolving these "legacy systems" is a significant challenge, as they are often critical to business operations but difficult to change.

The Heterogeneity Challenge

Modern software rarely operates in isolation. It must run on a variety of devices (desktops, mobiles, servers) and be integrated with other systems across networks. Maintaining these distributed systems is complex.

The Delivery Challenge

Businesses operate in a fast-changing environment and require software to be delivered quickly to respond to new opportunities and market pressures. Balancing speed of delivery with quality and cost is a constant engineering trade-off.

Professional Ethics for Software Engineers

Software engineers have responsibilities to the engineering profession and to society. The IEEE-CS/ACM task force developed a code of ethics with 8 guiding principles.

Public & Client

  • Public Interest: Act consistently with the public interest.
  • Client and Employer: Act in a manner that is in the best interest of their client and employer, consistent with the public interest.

Product & Judgment

  • Product: Ensure that products and related modifications meet the highest professional standards possible.
  • Judgment: Maintain integrity and independence in their professional judgment.

Management & Profession

  • Management: Managers and leaders shall promote an ethical approach to the management of software development and maintenance.
  • Profession: Advance the integrity and reputation of the profession consistent with the public interest.

Colleagues & Self

  • Colleagues: Be fair to and supportive of their colleagues.
  • Self: Participate in lifelong learning and promote an ethical approach to the practice of the profession.

Key Takeaways

Software Engineering

An engineering discipline concerned with all aspects of software production, from specification to maintenance.

Software

A combination of programs, documentation, and configuration data.

Software Process

The core activities of specification, development, validation, and evolution (maintenance).

Software Attributes

The essential qualities of good software: maintainability, dependability, efficiency, and usability.
