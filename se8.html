Advanced Process Models

Exploring Modern Software Development Methodologies

The Rational Unified Process (RUP)

A Framework for Iterative Development

RUP is a modern process model derived from the work on UML. It is an adaptable framework, intended to be tailored by development organizations. It is typically described from three perspectives:

  • Dynamic Perspective (Phases): Shows how the project progresses over time.
  • Static Perspective (Workflows): Shows the process activities or disciplines involved.
  • Practice Perspective: Suggests good practices that should be used throughout the process.

Dynamic Perspective: The Four Phases

  1. Inception: Establish the business case for the system. (Is this project feasible and worthwhile?)
  2. Elaboration: Develop an understanding of the problem domain and establish the system architecture.
  3. Construction: System design, programming, and testing. The bulk of the development work happens here.
  4. Transition: Deploy the system to its end-users.

Practice Perspective: RUP Good Practices

  • Develop software iteratively
  • Manage requirements systematically
  • Use component-based architectures
  • Visually model software (using UML)
  • Continuously verify software quality
  • Control changes to software

Static Perspective: Workflows (Disciplines)

The RUP framework identifies several key workflows. The effort dedicated to each workflow varies depending on the current phase of the project.

WorkflowDescription
Business ModelingModel the business processes using use cases.
RequirementsIdentify actors and develop use cases to model system requirements.
Analysis and DesignCreate architectural, component, and object models for the system.

Agile Methods

Responding to Change

Agile methods were created out of dissatisfaction with the overheads of heavyweight, design-heavy processes. They are best suited for small to medium-sized projects where requirements are expected to change.

Core Characteristics:

  • Focus on the code rather than extensive upfront design.
  • Based on an iterative approach to development.
  • Intended to deliver working software quickly and evolve it to meet changing requirements.
Agile PrincipleDescription
Customer InvolvementThe customer is closely involved throughout, providing and prioritizing requirements and evaluating iterations.
Incremental DeliveryThe software is developed in small increments, with the customer specifying requirements for each.
People, Not ProcessThe skills of the team are recognized and exploited. Teams develop their own ways of working.
Embrace ChangeExpect requirements to change and design the system to accommodate this.
Maintain SimplicityFocus on simplicity in both the software and the development process. Actively work to eliminate complexity.

Extreme Programming (XP)

An 'Extreme' Agile Approach

XP is one of the best-known and most widely used agile methods. It takes certain agile practices to 'extreme' levels.

  • New versions may be built several times per day.
  • Increments (small releases) are delivered to customers every 2 weeks.
  • Requires full-time engagement of an on-site customer representative.

Key XP Practices

  • Incremental Planning: Requirements are captured as 'user stories' on cards.
  • Small Releases: The smallest useful set of functionality is developed and released first.
  • Simple Design: Do just enough design to meet the current requirements and no more.
  • Test-First Development: Write an automated test *before* writing the code to implement a feature.
  • Refactoring: Continuously improve the internal structure of the code without changing its external behavior.
  • Pair Programming: Developers work in pairs, checking each other's work in real-time.
  • Collective Ownership: Anyone can change any part of the code. This spreads knowledge and prevents information silos.
  • Continuous Integration: Integrate and test code into the whole system as soon as a task is complete.
  • Sustainable Pace: Overtime is discouraged as it leads to burnout and reduced quality.
  • On-Site Customer: The customer is a full-time member of the development team.

Scrum

A Framework for Managing Complex Work

Scrum is an iterative and incremental agile framework for developing products or managing any work. It is named after the rugby formation where a group works together to move the ball forward.

Scrum Phases & Processes:

  • Pregame (Planning): Identify the project scope, prioritize initial requirements (in the Product Backlog), and establish the target environment.
  • Game (Sprints): The core of Scrum. Development happens in time-boxed iterations called Sprints, which typically last 1-4 weeks. Each Sprint includes analysis, design, development, and testing.
  • Postgame (Closure): At the end of a Sprint, a functioning increment of the product is delivered. The team conducts a review and prepares for the next Sprint.

Scrum Roles & Values

Scrum emphasizes team empowerment and collaboration. Key roles include the Scrum Master (facilitator), Developer, QA, etc. Core values are flexibility, frequent reviews, collaboration, and adaptability.

Process Improvement

The Improvement Cycle

Process improvement involves understanding existing processes and introducing changes to improve product quality, reduce costs, or accelerate schedules. It follows a continuous cycle:

  1. Process Measurement: Attributes of the current process are measured to establish a baseline.
  2. Process Analysis: The current process is assessed to identify bottlenecks and weaknesses.
  3. Process Change: Changes are introduced to address the weaknesses found during analysis.

The CMMI Framework

The Capability Maturity Model Integration (CMMI) is a widely used framework for process assessment and improvement. It provides a structured way for organizations to benchmark and improve their processes.

CMMI Staged Model Levels:
  1. Initial: Processes are chaotic and ad-hoc.
  2. Managed: Basic project management processes are established.
  3. Defined: Processes are documented, standardized, and integrated into an organizational standard.
  4. Quantitatively Managed: The organization and projects establish quantitative objectives for quality and process performance.
  5. Optimizing: The organization focuses on continual process improvement through quantitative feedback and piloting innovative ideas.
